Verification Checklist

Verify the following requirements are met:

  • Requirement 6.1: Segmented control displays as pill-shaped toggle for switching between tabs
  • Requirement 6.2: Active tab has white background (Latte) or Stone-700 background (Espresso) with subtle shadow
  • Visual: Container has warm gray background (--bg-secondary)
  • Visual: Tabs have 4px gap between them
  • Visual: Active tab has elevated appearance with box-shadow
  • Visual: Hover state shows subtle background change
  • Visual: Border radius is 12px for container, 8px for tabs
  • Behavior: Only one tab can be active at a time
  • Behavior: Tab content fades in smoothly when switching
  • Responsive: On mobile (<768px), tabs scroll horizontally if needed

Design System Tokens Used

The segmented control uses the following design tokens:

  • --bg-secondary - Container background (warm gray)
  • --bg-surface - Active tab background (white in Latte, Stone-800 in Espresso)
  • --bg-hover - Hover state background
  • --border-subtle - Container border
  • --text-primary - Active tab text
  • --text-secondary - Inactive tab text

Testing Instructions

  1. Click each tab and verify only one is active at a time
  2. Hover over inactive tabs to see the hover state
  3. Toggle between Latte and Espresso modes using the button above
  4. Resize the browser window to test mobile responsiveness
  5. Verify the pill shape is maintained in both themes
  6. Check that the active tab has a subtle shadow effect
